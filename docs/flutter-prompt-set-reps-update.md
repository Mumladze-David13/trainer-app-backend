# Обновление API: индивидуальные повторения по подходам (коммит c766293)

---

## Что изменилось

В упражнениях тренировки добавлено опциональное поле `setReps` — массив повторений для каждого подхода отдельно.
Работает аналогично уже существующему `setWeights`.

Затронуты два эндпоинта:
- `POST /workouts` — создать тренировку
- `PUT /workouts/:id` — обновить тренировку

---

## Новое поле setReps

| Поле | Тип | Обязательное | Описание |
|------|-----|-------------|----------|
| `reps` | `number` | да | Дефолтное кол-во повторений для всех подходов |
| `setReps` | `number[]` | нет | Повторения по каждому подходу отдельно |

Если `setReps` передан — он имеет приоритет над `reps` на каждый конкретный подход.
Длина массива `setReps` должна совпадать с `sets`.

---

## Примеры

### Одинаковые повторения (старое поведение, без изменений)
```json
{
  "exerciseId": "uuid",
  "sets": 3,
  "reps": 10,
  "weight": 80
}
```
→ 3 подхода по 10 повторений с весом 80 кг

---

### Разные повторения на каждый подход
```json
{
  "exerciseId": "uuid",
  "sets": 3,
  "reps": 10,
  "setReps": [12, 10, 8]
}
```
→ Подход 1: 12 повторений, подход 2: 10, подход 3: 8

---

### Разные веса И разные повторения (пирамида)
```json
{
  "exerciseId": "uuid",
  "sets": 4,
  "reps": 10,
  "setWeights": [50, 60, 70, 80],
  "setReps": [15, 12, 10, 8]
}
```
→ Подход 1: 15 × 50 кг, подход 2: 12 × 60 кг, подход 3: 10 × 70 кг, подход 4: 8 × 80 кг

---

## Ответ сервера

В объекте `WorkoutExercise` добавлено поле `setReps`:

```json
{
  "id": "uuid",
  "exerciseId": "uuid",
  "sets": 3,
  "reps": 10,
  "weight": null,
  "setWeights": null,
  "setReps": [12, 10, 8],
  "supersetGroup": null,
  "supersetOrder": null,
  "order": 0,
  "isDone": false
}
```

Если `setReps` не задан — поле будет `null`.

---

## Что делать во Flutter

**Модель `WorkoutExercise`** — добавить поле:
```dart
final List<int>? setReps;
```

**При отображении подхода** — если `setReps != null`, брать `setReps[i]` как количество повторений для i-го подхода; иначе использовать `reps`.

**При создании/редактировании** — если тренер задал разные повторения, передавать `setReps` как массив; если одинаковые — передавать только `reps`, `setReps` не передавать.
